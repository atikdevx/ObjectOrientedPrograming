## Description

**ObjectOrientedPrograming** is a structured collection of **Object-Oriented Programming (OOP)** implementations primarily focused on **Java**, alongside foundational examples in **C and C++**. It is created for deep learning, university revision, and technical interview preparation.

The repository is organized in a topic-wise manner so that each core pillar of OOP can be studied independently and revisited easily. It bridges the gap between theoretical definitions and practical software engineering patterns, helping you learn how to architect code based on real-world objects.

This repository is useful as:

- 📖 a revision guide for university exams and vivas
- 💡 a reference for understanding the "why" behind OOP design
- 🧠 a practice hub for transitioning from procedural to object-oriented thinking
- 🗂️ a topic-wise notebook for organized learning

## What’s Inside

### Java / Classes & Objects
The `Classes` folder contains foundational concepts of object creation and memory allocation. It covers:
* Defining blueprints (state and behavior)
* Object instantiation
* Understanding instance variables vs. `static` members

This section is great for building the absolute basics of grouping data and methods into cohesive units.

---

### Java / Constructors
The `Constructors` folder focuses on controlling the birth and initialization of objects, including:
* Default constructors
* Parameterized constructors
* Constructor overloading
* Understanding the `this` keyword

It is a solid reference for understanding object lifecycle and memory preparation.

---

### Java / Inheritance
The `Inheritance` folder explores code reusability and hierarchical relationships (`IS-A` relationships):
* Single Inheritance
* Multilevel Inheritance
* Hierarchical Inheritance
* Using the `super` keyword to chain parent logic
* *Note: Explains why Java rejects Multiple Inheritance of classes.*

This folder is ideal for learning how to eliminate code duplication by extending base logic.

---

### Java / Polymorphism
The `Polymorphism` folder breaks down the concept of "one name, multiple forms":
* **Compile-Time:** Method Overloading (changing method signatures).
* **Run-Time:** Method Overriding (Dynamic Method Dispatch) and the `@Override` annotation.

This section is critical for understanding upcasting and writing highly flexible, scalable code.

---

### Java / Abstraction & Interfaces
These folders focus on hiding complexity and defining strict architectural contracts:
* **Abstraction:** Abstract classes containing both concrete and abstract methods.
* **Interfaces:** Achieving total abstraction and multiple inheritance of type. 

This is the perfect place to learn how to design systems like `PaymentProcessor` that require standard behaviors across different classes.

---

### Java / Practice Programs
The `Practice Programs` folder combines all the above concepts into interactive systems:
* Fully functional, console-based, menu-driven programs.
* Mini-applications that demonstrate real-world modeling.

---

### C & C++
The `C` and `Cpp` folders look at OOP from a different lens:
* **C++:** Pointers, destructors, memory management, and virtual functions.
* **C:** Simulating object-oriented paradigms using Structs and Function Pointers.

## How to Use

1. Open the folder for the concept you want to study (e.g., `Java/Polymorphism`).
2. Read the internal documentation and code comments.
3. Compile the file using the appropriate compiler.
4. Modify the methods and test with your own real-world analogies.
5. Combine concepts in the `Practice Programs` directory.

Example for Java:

```bash
javac Main.java
java Main
